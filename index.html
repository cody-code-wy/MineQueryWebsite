MineQuery

What is MineQuery

Minequery is a JQuery inspired scripting platform for Minecraft, using Javascript. Intended for adventure maps, minigames, and modpacks. This allows creators to easily add an extra dimension to their creations. The main benefit to MineQuery is that it does not require extensive programming experience to use effectively.

What can it do?

Right now not very much, it is only a very early tech demo

Help! It crashed my game!

This tech demo is not stable! Bug reports are welcome, please email the crash logs, and copies of all scripts you are using, to bugreports@mineovation.com

How do I use it

Scripts are loaded from the minequery-scripts folder inside of the config folder for your Minecraft installation

API Documentation

Usage

MineQuery's featurs are exposed through the global function '$', this takes one argument: a selector. The selector determines what blocks/worlds/players/etc are affected by any chained functions.

Examples

Write Only Challenge

This code sample create a challenging gamemode that only lets players break blocks from worldgen, once they place a block down it is permanent. 


            $('*').on('place', function (event) {    // Whenever any block is placed
              $(this).on('break', function (event) { // when anyone tries to break that block
                event.cancel()                       // don't allow the block to break.
              }
            }

Selectors

  • Greedy Selector

    '*' Selects everything.

    Example 

    $('*')

    selects everything in game.

    Selector Type

    • Block:(ambiguous)
  • Mod Selector

    '@MODID' Selects all blocks/items from MODID.

    Example: 

    $('@minecraft')

    selects all blocks/items from vanilla minecraft.

    Selector Type

    • Block:(ambiguous)
    • Item:(ambiguous)
  • Block Name Selector

    '@MODID:BLOCKNAME' Select BLOCKNAME (every meta value) from MODID.

    Example: 

    $('@minecraft:wool')

    selects every color of wool.

    Selector Type

    • Block:(ambiguous)
  • Block Name Meta Selector

    '@MODID:BLOCKNAME:META' Select BLOCKNAME with meta META from MODID.

    Example: 

    $('@minecraft:dirt:1')

    selects all Coarse Dirt.

    Selector Type

    • Block:(ambiguous)
  • Location Selector

    '[X,Y,Z]' Select block located at X Y Z.

    Example: 

    $('[10,50,14]')

    selects block at x:10 y:50 z:14.

    Selector Type

    • Block:(disambiguous)
    • Entity:(disambiguous)
  • Range

    '[X1,Y1,Z1]:to[X2,Y2,Z2]' Selects all blocks between X1 Y1 Z1 and X2 Y2 Z2.

    Example: 

    $('[0,0,0]:to[100,100,-100]')

    selects all blocks from X:0 Y:0 Z:0 to X:100 Y:100 Z:-100.

    Selector Type

    • Block:(disambiguous)
    • Entity:(disambiguous)
  • Relative Range

    '[X1,Y1,Z1]:to[~,~+1,~-1]' selects all blacks between X1 Y1 Z1 and X1 Y1+1 Z1-1,

    Example: 

    $('[0,0,0]:to[~,~+256,~]')

    Selects from X:0 Y:0 Z:0 to build limit.

    Selector Type

    • Block:(disambiguous)
    • Entity:(disambiguous)

Functions

  • on 
    $('*').on('event_name', function (event) { //Callback })

    Registers a callback function to be called when the event with 'event_name' is fired and matches the selector.

    Example: 

    $(@minecraft:dirt).on('break', function (event) { console.log(this) })

    Registers callback to be called every time any dirt is broken that will print out information about the block broken.

    'on' works with any block selector.

Events

Event Functions

event listener callbacks will include an event object, these functions can be called on the event object. 

  • cancel()

    Will cancel event if possible 

  • uncancel()

    Will stop event from being canceled if possible

Event Types

These are the minecraft events that can hand event listeners registered to, these should be passed as the first argument in the 'on()' function

  • Block Break

    'break' Fires when a block is broken

    Applicable selectors

    • Block

    This selector: Selects block that is broken.

  • Block Place

    'place' Fire when a black is places

    Applicable selectors

    • Block

    This selector: Select block that was placed.
